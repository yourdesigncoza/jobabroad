# Stage 4.5 — Concatenate Sections

**Entry condition:** `current_stage: 4.5` (auto-advancing from stage 4).
**Purpose:** Assemble the 6 section files into the final `docs/guides/<category>.md` via atomic write.

Record stage start time for RUN_REPORT.

## 1. Mark in-progress

Update state:
```yaml
current_stage: 4.5
stage_status: in_progress
last_run_at: <now>
```

## 2. Verify all section files

Read each file listed in `state.section_files`. Check all 4 conditions:
- File exists on disk
- File is > 1000 bytes
- File starts with `## ` (an H2 heading)
- File contains at least one `<!-- src:` claim marker

If any section file fails any check:
```yaml
stage_status: failed
last_error: "Section file <path> failed check: <which check>"
last_error_class: output_invalid
```
Append to RUN_REPORT (failure block). Hard-exit:
```
✗ Stage 4.5 failed: section file <path> is missing or malformed (<check>).
Re-run stage 4 first: set current_stage to 4 and stage_status to failed in the state file,
then run /build-pathway <category>.
```

## 3. Assemble the guide (atomic write)

### 3a. Build content in memory

Build the final guide string by concatenating:

1. **Header block** (generate fresh):
   ```markdown
   <!-- Generated by build-pathway skill. Category: <category>. Assembled: <datetime>. -->
   ```
2. **Section 1** — full text of `docs/guides/<category>.section-01-destinations.md`
3. Blank line separator
4. **Section 2** — full text of `docs/guides/<category>.section-02-documents.md`
5. Blank line separator
6. **Section 3** — full text of `docs/guides/<category>.section-03-costs.md`
7. Blank line separator
8. **Section 4** — full text of `docs/guides/<category>.section-04-visa-routes.md`
9. Blank line separator
10. **Section 5** — full text of `docs/guides/<category>.section-05-scams.md`
11. Blank line separator
12. **Section 6** — full text of `docs/guides/<category>.section-06-contacts.md`

### 3b. Atomic write

Write to a temporary file first: `docs/guides/<category>.md.tmp`

Then rename/move to the final path: `docs/guides/<category>.md`

**Never write partial content to the canonical file.** If the write to `.tmp` fails:
```yaml
stage_status: failed
last_error: "Write to .tmp failed: <error>"
last_error_class: output_invalid
```
Append to RUN_REPORT (failure block). Hard-exit.

### 3c. Compute guide_hash

After the rename succeeds, compute SHA256 of the final guide:
```bash
shasum -a 256 docs/guides/<category>.md   # or sha256sum on Linux
```

Write to state: `guide_hash: "<64-char hex>"`

If hash computation fails:
```yaml
stage_status: failed
last_error: "SHA256 computation failed after write"
last_error_class: output_invalid
```
Append to RUN_REPORT. Hard-exit.

## 4. Verify the final guide

Read `docs/guides/<category>.md` back and confirm:
- File size > 8000 bytes
- Contains exactly 6 H2 headings (`## `)
- All 6 H2 headings match the canonical headings from guide-template.md:
  - `## 1. Destination Options — Where Can I Actually Go?`
  - `## 2. Document Checklist — What Papers Do I Need?`
  - `## 3. Realistic Costs — How Much Will This Actually Cost Me?`
  - `## 4. Visa Route Overview — What's the Actual Process?`
  - `## 5. Scam Red Flags — Will I Get Scammed?`
  - `## 6. Legitimate Contacts — Who Do I Actually Call?`
- No `{{PLACEHOLDER}}` text remains
- No section is empty (each H2 is followed by at least 200 characters before the next H2)

If any check fails:
```yaml
stage_status: failed
last_error: "Guide verification failed: <which check>"
last_error_class: output_invalid
```
Append to RUN_REPORT. Hard-exit with specific error.
