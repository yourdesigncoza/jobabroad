## Finding email addresses (do this FIRST)

Before treating any recipient as form-only, run a Google search for the domain. Use this exact query format:

```
email address for <domain>
```

Google's AI Overview pulls addresses from PDFs, JIRA-style help articles, image footers, and pages that defuddle-based crawlers can't reach. About 30 seconds per recipient.

**Observed hit rate on this batch:** 3 of 3 recipients (MMA Healthcare, Eteach, ECS Card) returned a usable address via this method after a defuddle crawl had missed it. That's a 100% rescue rate on the recipients where it was tried.

Apply this before adding any recipient to a form-only pile. Form submission takes roughly 5x longer than an email and doesn't build sender-domain reputation, so an email-find is high-value.

When you find an address by hand:

1. Run a one-off gog draft create command (see "Single-shot drafts without the script" below)
2. Remove the recipient from `submit-form.sh` so we don't double up
3. Update the Status log row above with the address and how it was found

## Drafting outreach with `gog` (CLI)

`gog` is the [Google Workspace CLI from gogcli.sh](https://gogcli.sh/). It is used to bulk-create Gmail drafts so each outreach batch doesn't require copy-pasting into the Gmail compose window. Drafts are reviewed and sent manually from Gmail; the CLI only handles draft creation, never sending.

### One-time setup (already done 2026-05-11)

- Binary installed at `~/.local/bin/gog` (`v0.16.0` Linux amd64 release).
- OAuth client registered: Google Cloud project `gog-cli`, Desktop OAuth client, Gmail API enabled, test user `laudes.michael@gmail.com` whitelisted on the consent screen (Testing mode).
- Encrypted file keyring at `~/.config/gogcli/keyring/`.
- Persistent env vars in `~/.bashrc`:
  ```bash
  export GOG_KEYRING_BACKEND=file
  export GOG_KEYRING_PASSWORD='jmkl'
  export GOG_ACCOUNT=laudes.michael@gmail.com
  ```

### Refresh-token expiry

The OAuth app is in **Testing** mode, which means the refresh token expires every **7 days**. When `gog auth doctor --check` reports a stale token (or any `gog gmail …` command fails with an auth error), re-run:

```bash
gog auth add laudes.michael@gmail.com --services gmail --force
```

A browser tab opens, you grant scopes again, and the new token is saved. About 30 seconds end to end.

To stop the weekly re-auth, publish the OAuth app in Google Cloud Console (Audience → Publishing). That involves a short verification flow for Google but gives long-lived refresh tokens. Not worth doing until weekly re-auth becomes annoying.

### Adding more drafts to the script

The script lives at `scripts/outreach-drafts/create-drafts.sh`. To add a new recipient:

1. Write the body to `scripts/outreach-drafts/bodies/<NN>-<slug>.txt`. One file per recipient. Use the same voice rules as the markdown drafts (no em-dashes, no intensifiers like "very").
2. Add a `create_draft` call to the script:
   ```bash
   create_draft \
     "#NN Recipient Name" \
     "to-address@example.com" \
     "cc-address@example.com" \
     "Subject line goes here" \
     "$BODIES_DIR/NN-slug.txt"
   ```
   Leave the CC argument as `""` (empty string) if there's no CC.
3. Dry-run first to confirm: `./create-drafts.sh --dry-run`.
4. When the dry-run output looks right, real-run: `./create-drafts.sh`.

Each invocation creates a **new** draft per recipient. Re-running the same script creates duplicates. If you accidentally do that, delete the extras in Gmail.

### Single-shot drafts without the script

For a one-off draft, call `gog` directly:

```bash
gog gmail drafts create \
  --from 'info@devai.co.za' \
  --to 'someone@example.com' \
  --cc 'optional@example.com' \
  --subject 'Subject line' \
  --body-file /path/to/body.txt \
  --plain
```

Useful flags:

- `--dry-run` — preview only, no API call.
- `--body` instead of `--body-file` — inline body string (handy for short messages).
- `--reply-to` — set a Reply-To header different from From.
- `--reply-to-message-id` and `--quote` — to draft a reply that quotes the original.

### Useful gog commands for outreach workflow

```bash
# List recent drafts (sanity check after running the script)
gog gmail drafts list --max 20 --plain

# Search inbox for outreach replies
gog gmail search 'from:globalnurseforce.com OR from:bic-immigration.com OR from:moveup.co.za' --max 10 --plain

# Read a specific message (sanitised body, safe to feed to AI)
gog gmail get <messageId> --sanitize-content --plain

# Verify send-as aliases (info@devai.co.za should appear)
gog gmail settings sendas list --plain

# Check auth health
gog auth doctor --check
```

### Safety notes

- The script never calls `gog gmail send` or `gog gmail drafts send`. It only creates drafts. Sending is always manual via Gmail.
- If you want belt-and-braces safety, add `--gmail-no-send` to any gog command to make it refuse to send mail even if you mistype.
- The encrypted file keyring stores a refresh token, not raw OAuth credentials. The credentials.json from Google Cloud Console lives separately at `~/.config/gogcli/credentials.json` (mode 0600). Both files together can authenticate as your Gmail account, so back them up but do not commit them to git.

### Files

- `scripts/outreach-drafts/create-drafts.sh` — orchestrator
- `scripts/outreach-drafts/bodies/*.txt` — one body file per draft (already committed; safe to share, no secrets)
- `~/.config/gogcli/credentials.json` — OAuth client (do not commit, not in this repo)
- `~/.config/gogcli/keyring/` — encrypted refresh token (do not commit, not in this repo)
