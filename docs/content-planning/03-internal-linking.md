# Internal Linking Map

Internal linking is what separates a topical-authority cluster from index bloat. Every
page must earn its place in the graph; an orphan route page is a thin page.

## The rule set

| From | Must link to | Why |
|---|---|---|
| Route `/routes/{role}/{country}` | Its category pillar `/pathways/{category}` | Up-link: passes context + authority |
| Route | The 3 SA document guides (police clearance, SAQA, apostille) | Reuse, not repetition (DRY) |
| Route | `/scam-warnings` (+ the route-specific scam passage) | Trust edge + conversion |
| Route | `/recruiters` (filtered to category where possible) | Commercial + trust |
| Route | The relevant comparison page (if one exists) | Lateral, decision-stage |
| Route | The free eligibility check (primary CTA) and R495 (secondary CTA) | Conversion |
| Pillar `/pathways/{category}` | Its 3–8 highest-priority routes | Down-link: distributes authority, makes routes discoverable |
| Blog post | 3–8 specific route pages it's relevant to | Authority layer feeds intent layer |
| Comparison page | Both single-route pages it compares | Resolves "X vs Y" into "now pick X" |

Targets: **5–8 internal links per route page**; **every route reachable from its pillar
in one click**; **no orphan routes**.

## Worked example — `/routes/registered-nurse/ireland/`

- ↑ `/pathways/healthcare`
- → `/guides/saqa-evaluation`, `/guides/police-clearance`, `/guides/apostille-dirco`
- → `/scam-warnings` (anchor: nursing-recruitment scams)
- → `/recruiters?category=healthcare`
- ↔ comparison: `/compare/nursing-uk-vs-ireland-south-africa`
- ↔ blog: `nursing-jobs-uk-from-south-africa`, `nurse-salary-uk-vs-south-africa`
- ⤓ CTA: free eligibility check + R495 plan

## Cannibalisation guard

Before publishing a route, confirm no existing blog already targets the same
`{role}+{country}` head term (see the de-dup table in `00-geo-strategy.md`). If one does,
the blog stays canonical for that term and the route either (a) is skipped, or (b) targets
a distinct transactional variant and cross-links. One primary keyword = one canonical URL.

## Implementation note (for when we build `/routes`)

The link rules are mechanical — they should be generated by the route handler from
`data/routes.csv` + the category map, not hand-authored per page. That keeps the graph
consistent and lets us add the document-guide / comparison links everywhere at once.
Mirror how `lib/pathway-content.ts` already centralises link handling.
